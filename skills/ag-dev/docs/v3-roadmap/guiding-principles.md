# Principios Guia V2

1. **Qualidade > Velocidade** — Melhor demorar e entregar codigo bom do que ser rapido e retrabalhar
2. **Protocolo > Convencao** — ACP garante que agentes se entendam, nao depende de "prompt engineering perfeito"
3. **Preservacao de Contexto** — Cada decisao, cada trade-off, cada debate fica registrado e consultavel
4. **Sob Demanda** — AG Dev nao e servico permanente, e uma ferramenta que voce liga quando precisa
5. **Clawdbot-First** — Tudo passa pelo Clawdbot como orquestrador principal, Claude Code e um canal de acesso
